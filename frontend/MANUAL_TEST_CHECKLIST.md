# Story 1.1-Frontend 手動測試清單

## 測試環境
- 前端: http://localhost:5174/register
- 後端: http://localhost:8000

## ✅ 測試項目

### 1. 頁面載入與顯示 (AC: 1, 9)
- [ ] 訪問 `/register` 路由可正常載入
- [ ] 頁面顯示所有必要欄位（顯示名稱、Email、密碼、確認密碼、服務條款）
- [ ] 響應式設計：手機 (375px)、平板 (768px)、桌面 (1024px+) 正常顯示
- [ ] Tailwind CSS 樣式正確應用

### 2. 表單驗證 (AC: 4)
- [ ] **顯示名稱驗證**
  - [ ] 少於 2 字元顯示錯誤
  - [ ] 超過 50 字元顯示錯誤
  - [ ] 包含非法字元（空格、特殊符號）顯示錯誤
  - [ ] 有效輸入（字母、數字、中文、底線）通過驗證
- [ ] **Email 驗證**
  - [ ] 無效格式顯示錯誤
  - [ ] 有效 Email 格式通過驗證
- [ ] **密碼驗證**
  - [ ] 少於 8 字元顯示錯誤
  - [ ] 缺少大寫字母顯示錯誤
  - [ ] 缺少小寫字母顯示錯誤
  - [ ] 缺少數字顯示錯誤
  - [ ] 符合所有條件通過驗證
- [ ] **確認密碼驗證**
  - [ ] 與密碼不一致顯示錯誤
  - [ ] 與密碼一致通過驗證
- [ ] **服務條款**
  - [ ] 未勾選無法提交表單
  - [ ] 勾選後可提交

### 3. 密碼強度指示器 (AC: 5)
- [ ] 輸入弱密碼顯示「弱」（紅色）
- [ ] 輸入中等密碼顯示「中」（黃色）
- [ ] 輸入強密碼顯示「強」（綠色）
- [ ] 指示器實時更新

### 4. 成功註冊流程 (AC: 7, 10)
- [ ] 填寫有效資料並提交
- [ ] 按鈕顯示 loading 狀態（spinner）
- [ ] 成功後顯示成功 toast 訊息
- [ ] 自動導向 `/dashboard`
- [ ] Token 正確儲存至 localStorage/sessionStorage
- [ ] 操作反饋在 300ms 內顯示

### 5. 錯誤處理 (AC: 8)
- [ ] **Email 已存在**
  - [ ] 註冊已存在的 Email
  - [ ] 顯示「Email 已被註冊」錯誤訊息
- [ ] **後端 API 錯誤**
  - [ ] 停止後端服務
  - [ ] 提交表單顯示網路錯誤訊息
- [ ] **表單驗證錯誤**
  - [ ] 提交無效表單
  - [ ] 顯示對應欄位的驗證錯誤

### 6. UI/UX 細節 (AC: 2, 6, 9)
- [ ] 所有輸入框有清晰的 label
- [ ] 錯誤訊息以紅色顯示在欄位下方
- [ ] 服務條款勾選框包含「服務條款」連結（指向 `/terms`）
- [ ] 「已有帳號？前往登入」連結指向 `/login`
- [ ] Focus 狀態有明顯視覺反饋
- [ ] 禁用狀態樣式正確

### 7. 無障礙性 (AC: 9)
- [ ] 所有表單元素有適當的 ARIA 標籤
- [ ] Tab 鍵可在欄位間導航
- [ ] Enter 鍵可提交表單
- [ ] 錯誤訊息有 `role="alert"` 屬性

### 8. 效能 (AC: 10)
- [ ] 首次載入時間 < 3 秒
- [ ] 按鈕點擊後立即顯示 loading 狀態
- [ ] 表單驗證反饋即時顯示
- [ ] 無明顯的 UI 延遲或卡頓

## 測試用例

### 有效註冊資料
```
顯示名稱: TestUser123
Email: test_user_new_$(date +%s)@example.com
密碼: Test1234
確認密碼: Test1234
服務條款: ✓
```

### 已存在的 Email（用於錯誤測試）
```
Email: test@example.com
```

### 無效資料範例
```
顯示名稱: a (太短)
Email: invalid-email (無效格式)
密碼: test (缺少大寫和數字)
確認密碼: different (不匹配)
```
